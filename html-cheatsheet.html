01 — HTML Basics

The foundation of every webpage. Learn the structure, syntax, and anatomy of HTML documents.

💡
What is HTML? HyperText Markup Language — it gives web content its structure. Browsers read HTML tags and render them as visual elements on screen.
Document Structure
Concept
Syntax / Code
Practice Tasks
HTML Boilerplate
The minimal skeleton every HTML file must have. DOCTYPE, html, head, and body are required.
Structure 
<!-- Every HTML file starts like this -->
<!DOCTYPE html>
<html lang="en">

  <head>
    <meta charset="UTF-8">
    <meta name="viewport"
      content="width=device-width">
    <title>Page Title</title>
  </head>

  <body>
    <!-- Your content goes here -->
  </body>

</html>
Write from scratch
  • Create a new file called index.html
  • Type the full boilerplate without copy-pasting
  • Change the title to your name
  • Open it in a browser
⌨️ vs code shortcut: type ! then press Tab
Tags & Elements
Tags are wrapped in angle brackets. Most come in pairs: opening tag and closing tag.
Structure 
<!-- Opening + Closing tag -->
<p>This is a paragraph.</p>

<!-- Self-closing (void) tag -->
<br>
<hr>
<img src="photo.jpg" alt="photo">
<input type="text">

<!-- Nesting (child inside parent) -->
<div>
  <p>I'm inside a div!</p>
</div>
Identify tags
  • Find 5 void (self-closing) tags in HTML
  • Write a div with 3 nested paragraphs
  • Spot the error: <p>Text</div>
Attributes
Attributes add extra info to tags. They go inside the opening tag as name="value" pairs.
Structure 
<!-- Attribute = name="value" -->
<a href="https://google.com">
  Go to Google
</a>

<!-- Multiple attributes -->
<img
  src="logo.png"
  alt="Company Logo"
  width="200"
  class="logo"
  id="site-logo"
>

<!-- Boolean attribute (no value needed) -->
<input type="checkbox" checked>
Apply attributes
  • Add id and class to a paragraph
  • Write an image tag with all 4 key attributes
  • Create a disabled input field
  • What is the difference between id vs class?
Comments
Comments are ignored by the browser. Use them to explain your code or temporarily hide elements.
Structure 
<!-- This is a single-line comment -->

<!--
  This is a
  multi-line comment
  Great for notes!
-->

<!-- Temporarily hide code: -->
<!--
<p>This won't show on screen</p>
-->

<p>This WILL show.</p>
Practice comments
  • Comment out a section of your HTML
  • Add author & date comments to a file
  • Use comments to label each section
⌨️ Shortcut: Ctrl + / (Win) / Cmd + / (Mac)

02 — Text Elements

Headings, paragraphs, inline formatting, quotes — everything for displaying text content.

⭐
SEO Tip: Each page should have exactly one <h1> tag. Headings h1–h6 create a content hierarchy that both users and search engines follow.
Concept
Syntax / Code
Practice Tasks
Headings h1–h6
Six levels of headings. h1 is most important, h6 least. Used for structure, not font size.
Text 
<h1>Main Title</h1>
<h2>Section Title</h2>
<h3>Sub Section</h3>
<h4>Sub Sub Section</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>
Build a hierarchy
  • Create a blog post structure with h1 → h3
  • Why should you NOT skip heading levels?
  • Style them with CSS to look the same — what changes?
Paragraph & Line Break
<p> creates a block of text. <br> forces a new line without a paragraph gap.
Text 
<p>This is a paragraph. It creates
space above and below automatically.</p>

<p>Line one.<br>
Line two — same paragraph.</p>

<!-- Horizontal rule (divider line) -->
<hr>
Write paragraphs
  • Write 3 paragraphs about yourself
  • Use <br> to write a 4-line poem in one <p>
  • When would you use <br> vs new <p>?
Bold, Italic, Underline
<strong> = important (bold). <em> = emphasis (italic). <u> = underline. Use semantic tags, not just visual ones.
Text 
<!-- Semantic (preferred) -->
<strong>Bold / Important text</strong>
<em>Italic / Emphasized text</em>

<!-- Presentational (avoid for meaning) -->
<b>Just visually bold</b>
<i>Just visually italic</i>
<u>Underlined text</u>

<!-- Can be combined -->
<strong><em>Bold AND italic</em></strong>
Format text
  • Write a sentence that uses strong, em, and u
  • Research: difference between <b> and <strong>
  • Nest bold inside italic and vice versa
Span & Div
<div> = block container. <span> = inline container. Both are generic with no visual meaning on their own.
Structure 
<!-- div = block level (takes full width) -->
<div class="card">
  <p>I'm inside a card div</p>
</div>

<!-- span = inline (sits in text flow) -->
<p>Price: 
  <span class="highlight">₹999</span>
   only!
</p>
Use containers
  • Wrap 3 elements in a div and add a border via CSS
  • Use span to color one word in a sentence
  • What happens when you put a div inside a span?
💡 div = layout container, span = text styling
Quotes, Code & Pre
Semantic tags for quotes, inline code snippets, and preformatted text that preserves whitespace.
Text 
<!-- Blockquote -->
<blockquote>
  "First, solve the problem. Then,
  write the code." — John Johnson
</blockquote>

<!-- Inline code -->
<p>Use <code>console.log()</code> to debug.</p>

<!-- Preformatted (preserves spaces/newlines) -->
<pre>
function greet() {
  return "Hello!";
}
</pre>
Quote and code
  • Add a famous programming quote using blockquote
  • Show a JS snippet using <pre><code>
  • Add <cite> to credit the quote author

03 — Links & Media

Anchor tags, images, audio, video — making HTML interactive and rich with media.

⚠️
Always use alt text! The alt attribute on images is required for accessibility and SEO. Screen readers use it. Search engines index it.
Concept
Syntax / Code
Practice Tasks
Anchor / Links
The <a> tag creates hyperlinks. href sets the destination. target controls where it opens.
Link 
<!-- External link -->
<a href="https://google.com"
   target="_blank"
   rel="noopener">
  Visit Google
</a>

<!-- Internal link -->
<a href="about.html">About Us</a>

<!-- Jump to section (ID) -->
<a href="#contact">Go to Contact</a>

<!-- Email link -->
<a href="mailto:hi@example.com">
  Email Me
</a>
Link everything
  • Create a nav bar with 4 internal page links
  • Link to a YouTube video with target="_blank"
  • Create a "Back to top" anchor link
  • Make an image clickable by wrapping <a> around <img>
Images
Display images with <img>. It is a void element — no closing tag. src and alt are required.
Media 
<!-- Basic image -->
<img
  src="photo.jpg"
  alt="A description of the image"
  width="400"
  height="300"
>

<!-- Image from URL -->
<img
  src="https://picsum.photos/400/300"
  alt="Random photo"
>

<!-- Figure with caption -->
<figure>
  <img src="chart.png" alt="Sales chart">
  <figcaption>Q3 Sales Data</figcaption>
</figure>
Display images
  • Add 3 images using picsum.photos URLs
  • Use <figure> + <figcaption> for a labelled photo
  • Break one image (wrong src) — what do you see?
  • Make an image link to another page

Video & Audio
HTML5 native media tags. No plugins needed. Support multiple file formats for browser compatibility.
Media 
<!-- Video -->
<video
  src="video.mp4"
  controls
  width="640"
  poster="thumbnail.jpg"
>
  Your browser does not support video.
</video>

<!-- Audio -->
<audio src="music.mp3" controls></audio>

<!-- Multiple sources fallback -->
<video controls>
  <source src="v.mp4" type="video/mp4">
  <source src="v.webm" type="video/webm">
</video>

Organize content with ordered, unordered, and definition lists. Display structured data with tables.

Concept
Syntax / Code
Practice Tasks
Unordered List

Text

  <li>Frontend

</ul>

Ordered & Description List
<ol> = numbered list. <dl> = definition list with terms and descriptions.
Text 
<!-- Ordered list -->
<ol type="1" start="1">
  <li>Learn HTML</li>
  <li>Learn CSS</li>
  <li>Build Projects</li>
</ol>
<!-- type: 1, A, a, I, i -->

<!-- Description list -->
<dl>
  <dt>HTML</dt>
  <dd>Structure of web pages</dd>
  <dt>CSS</dt>
  <dd>Styling of web pages</dd>
</dl>
Lists in practice

  • Make a glossary with <dl> for 4 HTML terms
  • Change an ol to use Roman numerals
Tables
Use <table> for tabular data. Never for layout. thead, tbody, tfoot organize rows semantically.
Table 
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Satish</td>
      <td>25</td>
      <td>Hyderabad</td>
    </tr>
  </tbody>
</table>

<!-- Merge cells -->
<td colspan="2">Spans 2 cols</td>
<td rowspan="3">Spans 3 rows</td>
Build tables

  • Use colspan to merge header cells
  • Style with CSS: border-collapse: collapse
  • Add a <caption> to describe your table

05 — Forms & Inputs

Collect user data with forms. The most interactive part of HTML — every input type explained.

🔗
Always link label to input! Use for="id" on labels and matching id on inputs. This is required for accessibility and clicking the label will focus the input.
Concept
Syntax / Code
Practice Tasks
Form Structure
The <form> wrapper. action = where data goes. method = GET or POST.
Form 
<form
  action="/submit"
  method="POST"
>
  <!-- Inputs go here -->
  <button type="submit">
    Submit
  </button>
</form>

<!-- GET = visible in URL (search forms)
     POST = hidden (login, sign-up) -->
Create a form
  • Build a contact form shell with no inputs yet
  • What is the difference between GET and POST?
  • What happens when you omit the action attribute?
Text Inputs
All the common text-based input types. type attribute controls keyboard and validation behavior.
Form 
<!-- Text -->
<label for="name">Name:</label>
<input type="text" id="name"
  name="name" placeholder="Your name">

<!-- Other types -->
<input type="email"  placeholder="Email">
<input type="password" placeholder="Password">
<input type="number" min="0" max="100">
<input type="tel"    placeholder="Phone">
<input type="url"    placeholder="Website">
<input type="date">
<input type="color">
<input type="range" min="0" max="10">

Select, Textarea & Checkboxes
Other essential form elements for choices, multi-line text, and multiple selections.
Form 
<!-- Dropdown -->
<select name="city">
  <option value="">Choose city</option>
  <option value="hyd">Hyderabad</option>

06 — Semantic HTML

Tags that describe meaning — not just appearance. Essential for accessibility, SEO, and clean code.

🎯
Semantic HTML = better everything. Screen readers navigate by landmarks. Search engines rank semantic pages higher. Developers maintain semantic code faster. Never use <div> when a semantic tag fits.
Concept
Syntax / Code
Practice Tasks
Page Layout Tags
Structural semantic elements that define distinct regions of a webpage. Replace generic divs.
Semantic 
<header>
  <!-- Logo, nav, hero -->
</header>

<nav>
  <!-- Navigation links -->
</nav>

<main>
  <article>
    <!-- Self-contained content -->
  </article>
  <section>
    <!-- Thematic grouping -->
  </section>
</main>

<aside>
  <!-- Sidebar, related links -->
</aside>

<footer>
  <!-- Copyright, links -->
</footer>
Build a page layout
  • Build a full blog page with: header, nav, main, article, aside, footer
  • Check with browser DevTools Accessibility pane
  • Rewrite the same page using only <div>s — what is lost?
  • Research: article vs section vs div — when to use each?
Inline Semantic Tags
Tags that add meaning to words and phrases within paragraphs — go beyond bold/italic.
Semantic 
<!-- Mark / highlight -->
<p>Search result: <mark>HTML</mark> basics</p>

<!-- Small print -->
<small>© 2024 All rights reserved</small>

<!-- Strikethrough (deleted) vs inserted -->
<del>₹1999</del> <ins>₹999</ins>

<!-- Superscript / subscript -->
<p>E=mc<sup>2</sup></p>
<p>H<sub>2</sub>O</p>

<!-- Abbreviation -->
<abbr title="HyperText Markup Language">
  HTML
</abbr>

<!-- Time -->
<time datetime="2024-06-15">June 15</time>
Enrich your content
  • Build a product price card: original with <del>, sale with <ins>, <mark> for "SALE"
  • Write a chemistry formula using <sub>
  • Add <abbr> to at least 3 tech acronyms
  • Add <time> to an article publish date

07 — Meta & Head Tags

Everything inside <head> — invisible to users but critical for browsers, SEO, and social sharing.

🔍
Meta tags = SEO foundation. The description, og:title, and og:image tags directly affect how your page appears in Google results and when shared on WhatsApp, Twitter, or LinkedIn.
Concept
Syntax / Code
Practice Tasks
Essential Meta Tags
Core meta tags every page needs. charset, viewport, description, and title are the minimum set.
Meta 
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">

  <!-- Responsive viewport -->
  <meta name="viewport"
    content="width=device-width,
    initial-scale=1.0">

  <!-- Tab title -->
  <title>Page Title | Site Name</title>

  <!-- Search engine description -->
  <meta name="description"
    content="150-160 char page summary.">

  <!-- Author -->
  <meta name="author" content="Your Name">
</head>
Optimize your head
  • Add all 5 meta tags to your project
  • What happens without the viewport meta on mobile?
  • Write a 155-character meta description for your portfolio
Open Graph Tags
og: tags control how your page looks when shared on WhatsApp, Facebook, LinkedIn, Twitter.
Meta 
<!-- Open Graph (social media preview) -->
<meta property="og:title"
  content="My Awesome Page">
<meta property="og:description"
  content="Page description here">
<meta property="og:image"
  content="https://site.com/img.jpg">
<meta property="og:url"
  content="https://yoursite.com">

<!-- Link external CSS -->
<link rel="stylesheet"
  href="style.css">

<!-- Favicon -->
<link rel="icon"
  href="favicon.ico"
  type="image/x-icon">

<!-- Load Google Font -->
<link rel="preconnect"
  href="https://fonts.googleapis.com">
Social & assets
  • Add OG tags to a portfolio page
  • Test with opengraph.xyz in browser
  • Link an external CSS file to your HTML
  • Add a favicon (use any .ico from the web)
  • Load a Google Font in the <head>
